№1
В основе стандартной библиотеки лежат следующие концепции:
1)она содержит базовые алгоритмы
2)она способна осуществлять ввод/вывод, работу с файлами
3)она содержит контейнеры(хранение данных)
4)переносимость(работа на различных платформах)
5)компактность(эффективность)

№2
Системы контроля версий используются для:
1)создания резервных копий
2)учет изменений, внесенных в код/проект
3)создания иерархии пользователей
4)разделения ответственности между разработчиками

№3
Взаимодействие с Git состоит из следующих основных действий:
1)git init — создание репозитория
2)git add и git rm — индексация изменений
3)git status — проверка состояния проекта, измененных и не добавленных файлов, индексированных файлов
4)git commit — совершение коммита
5)git reset — возврат к определенному коммиту
6)git branch — создание, перечисление и удаление веток
7)git checkout — переключение между ветками, извлечение файлов
8)git merge — слияние веток, разрешение возможных конфликтов

№4
Отдельные ветки для разработки следует создавать:
1) если работа над проектом ведется несколькими людьми
2) для сохранения безопасной стабильной версии проекта

№5
В библиотеке chrono реализованы 3 сущности:
1)интервалы времени(некоторое количество периодов) – duration;
2)моменты времени(интервал времени, измеренный на каком-либо таймере, начиная с некоторой точки отсчета) – timepoint;
3)таймеры – clock.
